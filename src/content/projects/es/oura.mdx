<strong>Oura</strong> es un proyecto destinado a permitir la creación de una
tienda dedicada a la ropa deportiva. Permite la creación de productos,
categorías y ventas con un gran grado de personalización.

<hr />

<h1 class="text-2xl">Tecnologías y Herramientas</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Next.js</h6>
    <p>
      Se eligió Next.js como framework para el proyecto debido a su sencillez en
      el desarrollo de componentes server-side y client-side, y su adopción en
      toda la industria. Este proyecto necesitaba mostrar una gran cantidad de
      datos, para lo cual la renderización server-side era una gran opción, y al
      mismo tiempo, necesitaba ser muy interactivo a medida que el usuario hacía
      clic en los productos o categorías. Debido a esto, Next.js fue una gran
      opción para este proyecto.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Stripe</h6>
    <p>
      Naturalmente, cada tienda debe tener un sistema de pago, y Stripe fue
      elegido como proveedor de pagos debido a su facilidad de uso y seguridad.
      Este proyecto fue la primera vez que usé Stripe y pude integrarlo con
      Next.js sin problemas.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">PostgreSQL</h6>
    <p>
      Se eligió PostgreSQL como base de datos debido a su confiabilidad y
      escalabilidad. Este proyecto tiene muchas partes relacionadas y era
      importante tener una base de datos que pudiera manejar todas estas
      relaciones e interacciones.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Roles y Responsabilidades</h1>
Este proyecto lo desarrollé yo solo en 2023. Fui responsable de cada faceta del
mismo, desde el diseño hasta el desarrollo. Cabe destacar:
<ul>
  <li>
    <h6 className="text-lg font-semibold">Desarrollador full-stack</h6>
    <p>
      Implementé la funcionalidad del proyecto y me aseguré de que fuera seguro
      y escalable. Utilicé Next.js, tRPC, NextAuth.js y Stripe, entre otras
      herramientas para desarrollar el proyecto.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Diseñador de UI/UX</h6>
    <p>
      Diseñé la interfaz de usuario y la experiencia de usuario del proyecto. El
      proyecto quería ser diferente de otras tiendas de ropa y yo quería crear
      una experiencia única para el usuario. Utilicé Figma para crear los
      diseños y los implementé en el proyecto usando TailwindCSS.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Desafíos y Soluciones</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Seguridad</h6>
    <p>
      Uno de los mayores desafíos al desarrollar un sitio web para una tienda es
      la seguridad.
    </p>
    <p>
      Para ayudar con esto, utilicé NextAuth.js, que proporciona una manera
      simple y segura de manejar la autenticación y autorización de usuarios.
      También utilicé Stripe para manejar los pagos y las funciones de seguridad
      de Supabase para garantizar que los datos estuvieran seguros.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">
      Implementación del sistema de pago
    </h6>
    <p>
      Esta fue la primera vez que trabajé con un sistema de pago y tuve que
      descubrir cómo integrarlo con el proyecto. Utilicé Stripe. Requirió un
      poco de configuración, pero fue bastante sencillo. Tuve que esforzarme un
      poco más para gestionar los pedidos, cómo etiquetarlos a medida que se
      creaban, cómo gestionar los eliminados, los reembolsos, los pagos
      fallidos, etc. Esto requirió mucha investigación y lidiar con casos borde.
    </p>
  </li>
</ul>
