<strong>Echoes</strong> es un proyecto que pretende ser una recopilación de
textos en un único lugar. Siempre he imaginado una forma diferente de encontrar
y financiar textos que se adapte a la era online, y creo que un sitio en el que
las partes principales de una historia estén disponibles de forma gratuita y los
lectores puedan pagar por los extras sería una gran idea.

<hr />

<h1 class="text-2xl">Tecnologías y Herramientas</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Astro</h6>
    <p>
      Este es un proyecto centrado en el contenido, por lo que Astro parecía la
      opción más adecuada. Astro tiene muchas características que hicieron que
      la creación de un sitio centrado en el contenido fuera muy sencilla.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Sanity</h6>
    <p>
      Como la idea era publicar nuevos textos con regularidad, decidí utilizar
      un CMS (sistema de gestión de contenido) para almacenar el contenido que
      luego se buscaría, en lugar de una base de datos normal o almacenarlo en
      el propio proyecto. Sanity es un CMS fácil de usar que se integra
      perfectamente con Astro gracias a su complemento oficial.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Roles y Responsabilidades</h1>
Este proyecto lo desarrollé yo solo en 2024. Fui responsable de cada faceta del
mismo, desde el diseño hasta el desarrollo. Cabe destacar:
<ul>
  <li>
    <h6 className="text-lg font-semibold">Gestión de contenido</h6>
    <p>
      Diseñé un sistema para almacenar y recuperar contenido usando Sanity, un
      CMS. Usando el cliente de Sanity, usé consultas GraphQL para recuperar el
      contenido y creé esquemas para los diferentes tipos de datos, como autores
      y textos.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Sistema de enrutamiento</h6>
    <p>
      Desarrollé el proyecto usando Astro y, entre otras tareas, desarrollé un
      sistema de enrutamiento que fuera intuitivo y claro para el usuario, donde
      cada pieza de contenido estuviera categorizada correctamente y las piezas
      relacionadas con ella fueran fácilmente accesibles.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Desafíos y Soluciones</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Implementación de Astro</h6>
    <p>
      Si bien ya estaba algo familiarizado con Astro, era la primera vez que lo
      usaba en un proyecto completo. Tuve que lidiar con el hecho de que la
      sintaxis de Astro es bastante diferente a la de otros frameworks, y tuve
      que aprender a usarlo de manera efectiva.
    </p>
    <p>
      Al mismo tiempo, tuve que lidiar con la integración de Sanity, ya que
      descubrí que usar las colecciones de contenido de Astro no sería óptimo
      para nuestro caso.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Implementación de Sanity</h6>
    <p>
      Sanity es un CMS relativamente fácil de usar y tiene buenas integraciones
      con Astro gracias a su plugin oficial. Sin embargo, tuve algunos problemas
      con la implementación del CMS, ya que era la primera vez que trabajaba con
      uno. Tuve que lidiar con el hecho de que el cliente de Sanity no era muy
      intuitivo, las queries eran muy largas y los datos no siempre eran fáciles
      de obtener.
    </p>
    <p>
      Para ayudar con esto, leí la documentación con atención y experimenté con
      diferentes queries para encontrar la mejor manera de obtener los datos que
      necesitaba.
    </p>
  </li>
</ul>
