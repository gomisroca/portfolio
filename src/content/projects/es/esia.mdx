<strong>Esia</strong> es un proyecto que pretende ser una galería de arte
online, donde los visitantes puedan ver las obras de arte y ver qué exposiciones
y eventos se están llevando a cabo en la galería.

<hr />

<h1 class="text-2xl">Tecnologías y Herramientas</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Next.js</h6>
    <p>
      Sabiendo que habría muchas obras de arte para mostrar, se decidió utilizar
      Next.js para crear el proyecto. Next.js es un framework que permite la
      renderización server-side, lo que es ideal para este proyecto, ya que
      permitiría tiempos de carga más rápidos y un mejor rendimiento.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">tRPC</h6>
    <p>
      Optamos por utilizar tRPC para gestionar la comunicación entre el cliente
      y el servidor, lo que garantizaría la seguridad de tipos y un desarrollo
      más rápido y seguro. Esta decisión se basó en el hecho de que esperábamos
      tratar con muchos tipos de datos complejos, y tRPC era una buena opción
      para esto.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Supabase</h6>
    <p>
      Supabase es una base de datos basada en la nube que proporciona una
      solución sólida y escalable para almacenar y recuperar datos. Con ella
      almacenamos todos los datos que necesitamos y utilizamos sus funciones de
      almacenamiento para almacenar las imágenes de las muchas obras de arte y
      exposiciones.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Vercel</h6>
    <p>
      Como decidimos usar Next.js, desplegar el proyecto en Vercel parecía la
      mejor opción y la más eficiente. Utilizamos Analytics y Speed Insights de
      Vercel para asegurarnos de que el proyecto se desarrollara sin problemas.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Roles y Responsabilidades</h1>
Este proyecto lo desarrollé yo solo en 2023. Fui responsable de cada faceta del
mismo, desde el diseño hasta el desarrollo. Cabe destacar:
<ul>
  <li>
    <h6 className="text-lg font-semibold">Diseñador de UI/UX</h6>
    <p>
      Diseñé una interfaz minimalista e intuitiva que permitiera una navegación
      sencilla sin distraer la atención de las obras de arte. Al ser una galería
      de arte, quería que el arte fuera el foco principal al visitar el sitio.
      Al mismo tiempo, utilicé animaciones al pasar el mouse y hacer clic para
      mostrar los datos de las obras de arte, lo que generó imágenes más grandes
      y atractivas.
    </p>
  </li>
  <li>
    <h6 className="text-lg font-semibold">Desarrollador full-stack</h6>
    <p>
      Desarrollé el proyecto usando Next.js y tRPC, conectándo a una base de
      datos PostgreSQL a través de Prisma. Desarrollé métodos para cargar
      imágenes en los Buckets de Supabase, para optimizar dichas imágenes para
      su visualización. Además, utilicé métodos como la paginación para
      asegurarme de que el sitio fuera rápido y respondiera a pesar de la gran
      cantidad de imágenes.
    </p>
  </li>
</ul>

<hr />

<h1 class="text-2xl">Desafíos y Soluciones</h1>
<ul>
  <li>
    <h6 className="text-lg font-semibold">Optimización de imágenes</h6>
    <p>
      Uno de los mayores problemas que enfrenté durante este proyecto fue
      optimizar las imágenes para su visualización. Con la cantidad de imágenes
      que tenía el proyecto, era importante evitar tiempos de carga lentos.
    </p>
    <p>
      Para ayudar con esto, utilicé esqueletos mientras se cargaba la imagen,
      paginación para evitar cargar demasiadas imágenes a la vez y lazy loading
      para cargar las imágenes solo cuando están a la vista.
    </p>
  </li>
</ul>
